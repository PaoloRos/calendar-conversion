AI tool usage
=============

This project was built with **OpenAI Codex GPT 5.6 Sol** as an AI-assisted
programming tool. The project owner remained responsible for the requirements,
decisions, and acceptance of the result.

The workflow was:

1. A brainstorming phase explored the application and its possible design.
2. The project owner supplied implementation instructions through ``TODO.md``,
   sometimes incorporating suggestions proposed by the AI tool.
3. Agreed technical choices were recorded in
   ``IMPLEMENTATION_DECISIONS.md``.
4. The AI tool implemented code and wrote automated unit tests.
5. The project owner verified that each part functioned correctly. Verification
   was practical rather than exhaustive, and included visual checks where
   presentation or calendar behavior mattered.

AI output was therefore treated as an implementation aid, not as an autonomous
source of product requirements or final approval.
